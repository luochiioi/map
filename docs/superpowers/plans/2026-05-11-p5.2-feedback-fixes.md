# P5.2 真机反馈修复与奖励记录 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复真机反馈中的地图 marker 点击、跨页聚焦、后台未登录卡死、地图图标遮挡，以及补齐后台奖励兑换记录查询。

**Architecture:** App 侧把“回首页并打开某 marker”抽成统一导航/聚焦 helper，所有入口写同一套 focus payload，首页统一消费；marker 点击问题先用纯函数和日志定位 markerId 归一化边界，再修复 `checkin-map` / `findById` 链路。后台侧新增 auth error CTA 和 rewards 管理页，复用 `rewards` 集合与 P5.1 的积分 helper。

**Tech Stack:** uni-app x / UTS 5.07、uniCloud 云对象、uni-admin Vue、Node `node:test`。

---

## 0. 当前反馈与判断

1. 后台新增打卡点可显示图标与标题，但点击 marker 只出现标题，不弹 App 详情面板；从任务页定位能打开详情，说明 `CheckinMarker` 数据和详情面板基本可用，风险点在 `markertap.detail.markerId` 到 `findById()` 的映射或 SDK marker id 边界。
2. 我的打卡页“在地图上查看”仍调用 `uni.switchTab({ url:'/pages/index/index' })`，但 App `pages.json` 没有 tabBar，必修。
3. 路线详情“去这里”当前 `navigateBack()` 会回到上一页。若上一页是 `/pages/routes/routes`，就会回路线列表而不是首页地图，必修。
4. 后台未登录时页面只显示错误，无登录按钮，用户会卡住。
5. 地图缩小时 marker 图标大小不变，视觉遮挡地图内容。优先采用“随 scale 分档调整 icon size + 使用更轻量 pin 图标”的方案，不做聚合点，避免 P5.2 过大。
6. 后台只在用户管理显示奖励计数，没有“奖励兑换记录”明细页。需要新增 admin rewards 查询 API + 页面入口。

- [ ] **Step 3: 实现 consumeFocusPayload**

```ts
export function consumeFocusPayload(): FocusPayload | null {
  const inMemory = focusPayload.value
  if (inMemory != null) {
    focusPayload.value = null
    uni.removeStorageSync(KEY_PENDING_FOCUS)
    return inMemory
  }
  const raw = uni.getStorageSync(KEY_PENDING_FOCUS)
  if (raw == null || typeof raw !== 'string') return null
  const parsed = JSON.parse<FocusPayload>(raw as string)
  uni.removeStorageSync(KEY_PENDING_FOCUS)
  return parsed
}
```

```ts
function normalizeMarkerIdLocal(value: any): number | null {
  if (value == null) return null
  if (typeof value === 'number') return value as number
  if (typeof value === 'string') {
    const raw = value as string
    if (raw.length === 0) return null
    const n = parseInt(raw)
    if (!isNaN(n)) return n
  }
  return null
}
```

- [ ] **Step 2: 实现 normalizeRewardRecords**

Use `buildUserLookup` style inside reward-service or pass a lookup:

```js
function normalizeRewardRecords(rewards, uniUsers) {
  const lookup = new Map()
  ;(uniUsers || []).forEach(user => {
    const uid = String(user && user._id || '')
    if (!uid) return
    lookup.set(uid, String(user.nickname || user.username || uid))
  })
  return (rewards || []).map(row => {
    const sourceType = row.source === 'route' || row.routeId != null ? 'route' : 'task'
    const rewardPoints = row.rewardPoints != null ? Number(row.rewardPoints) : parseRewardPoints(row.reward)
    return {
      _id: row._id,
      userId: row.userId || '',
      userName: lookup.get(String(row.userId || '')) || String(row.userId || ''),
      sourceType,
      sourceTitle: sourceType === 'route' ? String(row.routeName || '') : String(row.taskName || ''),
      reward: String(row.reward || ''),
      rewardPoints: Number.isFinite(rewardPoints) ? rewardPoints : 0,
      rewardClaimed: row.rewardClaimed === true,
      statusText: row.rewardClaimed === true ? '已兑' : '待兑',
      earnedAt: Number(row.earnedAt || 0),
      claimedAt: row.claimedAt != null ? Number(row.claimedAt) : null
    }
  })
}
```
